# 百工 Baigong — 更新日志

## v0.8.0 (2026-06-20)

### 🎯 纯 Python 重写！不再依赖 HTML/JS/WKWebView

**整个桌面 GUI 从 pywebview + HTML/CSS/JS 全面迁移到 Pure Python PySide6。**

### 为什么这么做？
- ✅ **修复启动卡加载** — WKWebView 渲染 HTML 的兼容性问题彻底消除
- ✅ 不再需要嵌入式 Web 服务器渲染 UI
- ✅ 原生 macOS Qt 控件 = 更快启动、更流畅交互
- ✅ Python 开发者可直接修改全部 UI 代码

### 变化
| 方面 | 旧版 | v0.8.0 |
|------|------|--------|
| 前端技术 | HTML+CSS+JS (482行) | **纯 Python PySide6** (1743行) |
| 窗口引擎 | pywebview (WKWebView) | **原生 Qt Widgets** |
| 渲染方式 | 浏览器渲染 HTML | Qt 引擎直接绘制控件 |
| 主题系统 | CSS 变量+JS切换 | **QSS 动态样式表** |
| 启动卡死 | ⚠️ 常见 | ✅ 已修复 |

### 功能全部保留
- 三栏布局（Agent 列表 | 看板/编辑 | 日志）
- 7 套完整主题色（靛蓝/琥珀/翡翠/暮紫/玫瑰/极夜/白昼）
- Agent 管理（创建/编辑/删除，独立 LLM 配置）
- 任务看板（待处理/进行中/已完成，进度条，工作流状态）
- 热更新（检查/下载/自动安装）

## v0.6.0 (2026-06-20)

### New — 自动化 Agent 工厂 🏭
- **主管 Agent**：启动时自动创建默认「主管」Agent，负责接收用户任务
- **自动任务拆解**：主管用 LLM 分析任务 → 自动拆解为可并行执行的子步骤
- **自动招聘**：主管根据拆解结果，自动创建合适的子 Agent（研究员、编辑、管理员等）
- **并行执行**：所有子 Agent 同步执行各自的子任务（asyncio.gather）
- **自动汇总**：所有子任务完成后，主管自动汇总最终报告
- **临时 Agent 标记**：系统自动创建的 Agent 显示 📎 标记和创建者（"由 主管 创建"）
- **任务父子关系**：根任务→子任务，看板完整展示任务层级

### Example 工作流程
用户下发"收集10篇翡翠资料并入库" →
1. 📋 主管拆解为3步：搜索 / 整理 / 入库
2. 🆕 自动创建 王研究员、李编辑、陈管理员
3. ⚡ 三人并行工作
4. 📊 主管汇总报告
