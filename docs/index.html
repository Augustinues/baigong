🏮 百工 — 系统架构

百工是一个自我进化的多 Agent 协作系统。每个 Agent 是独立运作的个体，通过消息总线和共享看板异步协作，就像公司里各司其职的工匠。

五大核心组件

1. Agent 注册表 — 每个 Agent 独立配置：角色、工具、模型（可混用不同厂商）、权限。支持 YAML 定义。

2. 消息总线 — Agent 之间通过消息通信，不是函数调用。支持单播、广播、多轮对话（thread_id）。

3. 任务看板 — 信息公开在任务卡片上，Agent 自己去板上取。主管在运行时动态拆解任务，不是预先画好的 DAG。

4. Agent 运行时 — 每个 Agent 运行一个独立的思考-行动循环：感知 → 思考(LLM) → 决策(权限检查) → 行动(工具/消息) → 更新状态

自我进化机制

记忆自动巩固: 任务完成后，Agent 自动判断哪些经历值得记住（CEO 反馈 0.9、失败 0.7、重复模式 0.4），用树+图结构持久化。

联想检索: 从入口节点沿树和链接 BFS 遍历，像人脑一样"想到翡翠就浮现鉴别→B货→证书"。

Skill 自动形成: 同类型任务做 3 次以上，自动从记忆树中挖掘可复用的工作流程。下次遇到同样任务跳过 LLM 思考直接执行。

五层容错: LLM 重试 → 死循环检测 → 任务恢复 → Agent 重启 → 系统崩溃恢复

技术栈

Python asyncio 事件循环 · SQLite 树+图记忆网络 · 每个 Agent 独立模型配置 · 纯接口设计（不绑定工具实现）
